## Commit Message Guidelines

### Commit Types

- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation changes
- `refactor:` for code restructuring
- `chore:` for maintenance tasks
- `test:` for test-related changes
- `ci:` for CI/CD related changes
- `perf:` for performance improvements
- `revert:` for reverting to a previous commit
- `style:` for formatting only changes
- `security:` for security fixes

### Common Scopes

- `flake.lock`
- `hosts/`

### Rules

- Subject line maximum length 50 characters.
- Capitalize the subject line and do not end it with a period.
- Separate the subject line from the body with a blank line.
- Keep commits focused on a single logical change whenever practical.

### Examples

feat(aerospace): add workspace rules

chore(lockfile): update flake.lock

revert(shell): remove broken zsh configuration
